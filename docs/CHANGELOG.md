# Changelog

Living log of significant changes to the project. This is **not** optional bookkeeping — every significant change (new feature, format change, dependency change, removed feature, policy change) gets an entry here at the time it's made. See the documentation policy in [.claude/CLAUDE.md](../.claude/CLAUDE.md).

Format: `YYYY-MM-DD — short description. Why (if not obvious). Files touched.`

## 2026-07-08

- Line tool follow-ups: Free/Smart routing modes, curve handle, multi-select, rubber-band and Delete-key support (PATCH-ish but bundled as part of the still-unreleased v0.3.0 Line feature above — no separate version bump). Toolbar tooltips for drag-only tools dropped the "— drag onto the board" suffix (redundant now that clicking shows the same hint). New lines and card-to-card arrows default to `arrow: true` (previously the standalone Line tool defaulted the arrowhead off). Edge selection changed from a single `BoardView.selectedEdge: string | null` to `selectedEdges: Set<string>`, so rubber-band selection and Delete now work on one or more lines at once (a line counts as "inside" the band only if both endpoints are). Board file format: new optional `Edge.mode` (`"free" | "smart"`, undefined = `"smart"` for backward compatibility; new lines are created with `"free"`) and `Edge.bend` (`{x,y}` world point) — a Free-mode line renders as a straight segment clipped to each end's boundary, optionally bowed through a draggable midpoint handle into a quadratic curve; Smart mode is the original auto-routed bezier. Right-click menu on a line gained a "Switch to Smart routing" / "Switch to Free line" toggle. Files: `src/types.ts`, `src/render.ts`, `src/board-view.ts`, `src/modals.ts`, `styles.css`, `docs/*`.
- Reorganized the main toolbar and added a standalone **Line** tool (v0.3.0, MINOR — new feature). The toolbar now has three groups separated by dividers: **draggables** (Note, To-do, Line, Column, Board, Swatch, Sketch, Comment), **flexible tools** (Image, Vault file, Link — all drag-only — and Draw, click-only), and **actions** (Undo, Redo, Shortcuts). Draggable tools are now **drag-only**: a plain click no longer creates anything — the button shakes and shows a hint tooltip prompting a drag. Their single-key shortcuts (`N`/`T`/`C`/`L`/`B`/`S`/`M`) were removed; only `D` (Draw) and `/` (Shortcuts) remain. The old two-click **connect mode** button (`spline`, `A`) was removed — the Line tool replaces it: drag Line onto the board to drop a free segment, then drag either endpoint handle onto a card to anchor it (drop on empty canvas to leave it free). The card-hover blue dot still draws an arrow, and now dropping it on empty canvas leaves a free endpoint too. Board file format: `Edge.from`/`Edge.to` are now optional and a new optional `Edge.fromPt`/`Edge.toPt` (`{x,y}` world point) stores a free endpoint — backward-compatible (existing edges keep both anchors). Files: `src/types.ts`, `src/render.ts`, `src/board-view.ts`, `src/modals.ts`, `styles.css`, `package.json`, `manifest.json`, `versions.json`, `docs/*`.
- Added **Draw** and **Sketch** freehand drawing (v0.2.0, MINOR — new feature). Draw (toolbar, `D`) enters a board-wide draw mode: the main toolbar dims and a reusable **contextual toolbar** (pen / select / eraser | color / stroke size | undo / redo | discard / save) appears beside it, over a dimmed scrim that makes board nodes inert. Strokes support pen pressure (`PointerEvent.pressure`). On save, nearby strokes are clustered into `drawing` board items (gap threshold `DRAW_GROUP_DISTANCE`, tunable in `src/types.ts`); double-clicking a drawing re-opens it for editing. Sketch (toolbar) creates a `sketch` card whose double-click opens a popup canvas (like the vault-file preview) to draw inside; the card shows a scaled preview. Board file format: two new `Item.type`s (`drawing`, `sketch`) and an optional `Stroke[]` `strokes` field (points in item-local coords) — backward-compatible. New dependency `perfect-freehand@1.2.3` (stroke outlining, bundled). Files: `src/types.ts`, `src/draw.ts` (new), `src/board-view.ts`, `src/render.ts`, `styles.css`, `package.json`, `manifest.json`, `versions.json`, `docs/*`.

- Added a versioning policy: SemVer 2.0.0, where only shipping-plugin changes (`src/**`, `styles.css`, user-facing `manifest.json` fields) bump the version and docs/tooling/dev-dependency changes do not. Documents MAJOR/MINOR/PATCH rules, the pre-1.0.0 exception, the Obsidian strict-`x.y.z` constraint on `manifest.json`/`versions.json`, and the three version files to keep in sync. Files: `.claude/instructions/versioning.instructions.md` (new), `.claude/CLAUDE.md`.
- Added optional auto-copy step to `npm run build`: after a successful build, `scripts/copy-to-vault.mjs` copies `main.js`, `manifest.json`, `styles.css` into a vault plugin folder read from a gitignored `.env.local` (`OBSIDIAN_PLUGIN_DIR=...`). No-op if `.env.local` is absent. Files: `scripts/copy-to-vault.mjs`, `package.json`, `.env.local` (untracked), `docs/DEVELOPMENT.md`.
- Fixed `docs/DEPENDENCIES.md`: pinned-version table and esbuild CVE note were stale (`package.json` had already moved to `esbuild@0.28.1`/`typescript@5.4.2` but the doc still said `0.20.0`/`5.4.0`). Updated table and dropped the now-inapplicable CVE mitigation reasoning since `0.28.1` is already >=0.25.0. Files: `docs/DEPENDENCIES.md`.
- Refactored `styles.css` to centralize repeated colors as CSS custom properties (`--mgn-*`) in a `:root` block, instead of the same hex/rgba values being copy-pasted across the toolbar, zoombar, search bar, crumbs, edges, and card states. Also fixed a visual inconsistency where the "Dark" card color preset looked different between note/comment cards and to-do cards: note placeholders and code/blockquote styling used hardcoded black-based `rgba()` values (readable only on light cards), while to-do inputs already used `color: inherit` + `opacity`. Switched the note-side rules to the same `inherit`/`color-mix(currentColor, ...)` approach so all card types stay legible across every color preset. No behavior change beyond the corrected contrast. Files: `styles.css`.
- Added line (arrow/connector) color. Right-click a line -> "Line color" submenu, reuses the same 10-color palette as cards. Stored as `Edge.color` (board file format addition, backward-compatible since it's optional). Files: `src/types.ts`, `src/render.ts`, `src/board-view.ts`.
