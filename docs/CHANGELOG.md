# Changelog

Living log of significant changes to the project. This is **not** optional bookkeeping — every significant change (new feature, format change, dependency change, removed feature, policy change) gets an entry here at the time it's made. See the documentation policy in [.claude/CLAUDE.md](../.claude/CLAUDE.md).

Format: `YYYY-MM-DD — short description. Why (if not obvious). Files touched.`

## 2026-07-11 (v0.8.0)

- Fixed the empty sketch card's "Double-click to draw" placeholder not being centered: horizontally, the generic `.mgn-placeholder` div shrinks to its text width so its flex parent couldn't center it as a block; vertically, `.mgn-sketch-preview` only centered within its own 120px min-height box, leaving dead space below when the card was resized taller (the card itself wasn't a flex column, so the preview never grew to fill it). Files: `src/render.ts`, `src/styles/drawing.css`, `styles.css`.
- Added a portable **template bundle** format and export/import for it. "Save current board as template" (and the new "Export current board as template" zoombar button) now walk the board recursively — every nested "board" card, plus every image/file/recording it references — and pack it all into one `<name>.board.template` JSON file (assets embedded as base64) saved to Settings → Templates folder, instead of the old plain-copy `.board` file that silently broke as soon as the board referenced anything outside itself. "New board from template" and the new "Import template" zoombar button both unpack a bundle (recreating every board/asset with fresh de-duplicated paths and rewriting `Item.path` references to match) — Import lands the result in the templates folder so it becomes reusable; New board from template lands it next to the active file and opens it, like before. Import always shows a warning modal (a template file can bundle files of any type) before writing anything, and can be cancelled. The import file picker is restricted to `.board.template`; on desktop it uses Electron's native open dialog defaulting to the templates folder (falls back to a plain `<input type=file>` if Electron's dialog isn't available, and always on mobile). New `src/template-bundle.ts` (`collectBundle`/`unbundleTemplate`), new `ImportTemplateConfirmModal` in `src/modals.ts`. Files: `src/template-bundle.ts` (new), `src/main.ts`, `src/board-view.ts`, `src/modals.ts`.

## 2026-07-11 (internal refactor, no behavior change)

- Split `src/board-view.ts` (2305 lines, 5x the next-largest module) into single-responsibility modules following the existing `render.ts`/`draw.ts` delegation convention (`export function xyz(view: BoardView, ...)`, with `BoardView` keeping its public method names as one-line delegates) — purely for maintainability, no behavioral or file-format change. New files: `src/geometry.ts`, `src/board-history.ts`, `src/board-camera.ts`, `src/board-interaction.ts`, `src/board-item-crud.ts`, `src/board-clipboard.ts`, `src/board-context-menu.ts`, `src/board-keyboard.ts`, `src/board-drop-import.ts`, `src/drawing-toolbar.ts`, `src/record-card.ts`, `src/file-preview.ts`, `src/board-search.ts`. Also split `styles.css` (1345 lines) into partials under `src/styles/` (`variables.css`, `layout.css`, `cards.css`, `edges.css`, `chrome.css`, `preview.css`, `drawing.css`, `index.css`), bundled back into the root `styles.css` by a second `esbuild.config.mjs` build step — the root `styles.css` is now a generated build artifact. Updated `build-local.mjs` (the no-npm fallback bundler) to add all new files to its concatenation list in dependency order, and to preserve renamed local imports (`import { x as y } from "./z"` now emits `const y = x;` instead of being silently dropped) since `BoardView`'s delegate methods import every extracted function under an aliased `...Impl` name. No version bump — this is an internal, non-shipping-behavior change. Files: `src/board-view.ts` + all new files above, `esbuild.config.mjs`, `build-local.mjs`, `docs/ARCHITECTURE.md`, `docs/DEVELOPMENT.md`.

## 2026-07-10 (v0.7.1)

- Fixed the "Default microphone" dropdown in Settings → Recording rendering oversized: a native `<select>` sizes itself to its widest *option* (not the selected one), and mic device labels can be long (e.g. "Microphone (Realtek High Definition Audio)"), unlike the Font dropdown's short labels. Reverted the earlier attempt that also resized the OpenAI API key field (that field's default Obsidian sizing was fine and shouldn't have been touched) — only `.mgn-mic-dropdown` gets the width cap now (`width/max-width: 250px !important; flex: 0 0 auto !important;`, targeting the Font dropdown's natural size). Files: `src/settings-ui.ts`, `styles.css`.

## 2026-07-10 (v0.7.0)

- Moved the OpenAI API key **out of the vault on desktop**: it was living in `data.json` inside `.obsidian/plugins/maguilanote/`, so any vault backup/sync would carry it along. New `src/secrets.ts` stores it as plain JSON at `~/.maguilanote/secrets.json` (OS user home, outside any vault; best-effort `chmod 600` on POSIX) and is used whenever `Platform.isDesktopApp` is true. This is "not backed up with your notes," not OS-keychain-grade encryption — still plaintext on disk. Mobile has no filesystem access outside the vault, so `MaguilanoteSettings.openaiApiKey` remains as a mobile-only fallback (still in `data.json`, still included in vault backups there — the Settings description now says so explicitly). New `MaguilanotePlugin.getOpenAiApiKey()` picks the right source; `BoardView.transcribeRecord` and the Settings UI both read/write through it instead of `settings.openaiApiKey` directly. Files: `src/secrets.ts` (new), `src/main.ts`, `src/settings-ui.ts`, `src/board-view.ts`.

## 2026-07-10 (v0.6.0)

- Added **"Transcribe text"** to a Record card's right-click menu (shown once it has a recording). Sends the audio file to the OpenAI Whisper API (`whisper-1`, REST `fetch`, no new npm dependency) using a new **OpenAI API key** setting (Settings → Recording, stored locally in `data.json`, only ever used for this call). On success, creates a new note card next to the recording with the transcribed text and connects the two with an arrow (`BoardView.transcribeRecord`). This is a paid, external, opt-in feature — nothing is sent anywhere unless the user pastes an API key and explicitly invokes it; without a key it shows a Notice pointing at the setting instead of failing silently. Files: `src/board-view.ts`, `src/main.ts`, `src/settings-ui.ts`.

Fixed the empty-record card's "Double-click to record" placeholder not being centered (it inherited the generic left-aligned `.mgn-placeholder` layout). ## 2026-07-10 (v0.5.0)

- Added a **Record** card: a new draggable toolbar tool that creates an audio-recording card. Double-clicking it opens a popup with a microphone dropdown, Record/Stop buttons, and a live timer; on save the clip is recorded via `getUserMedia`/`MediaRecorder`, written to the board's `assets/` folder as a `.webm` vault file (reusing the same binary-save pattern as pasted/dropped OS files), and the card then plays it back inline with `<audio controls>`, same as an audio "file" card. New `Item.duration` field (seconds) alongside the existing `path` field (reused for the audio file reference). New Settings → Recording section with a "Default microphone" dropdown (`MaguilanoteSettings.defaultMicId`, populated from `navigator.mediaDevices.enumerateDevices()`), used to pre-select the mic in the recording popup; the popup also lets you switch mic per-recording. Files: `src/types.ts`, `src/board-view.ts`, `src/render.ts`, `src/main.ts`, `src/settings-ui.ts`, `styles.css`.

## 2026-07-09 (follow-up 2)

- Second follow-up to the Settings panel, still v0.4.0 (no separate bump):
  - **Removed the text-size setting entirely** (didn't look good) — reverted `--mgn-font-scale`, the slider, and all the `calc(... * Npx)` font-size wrapping in `styles.css` back to plain pixel values. Kept the `.mgn-note-body` heading-size fix from the previous follow-up (still correct on its own, independent of scaling) but reworded its comments since they no longer reference a scale setting.
  - Added **per-theme color customization** to Settings → Customization: board background, default card background, and each of the 8 named card colors (Yellow/Orange/.../Gray), each with its own color picker. Colors are stored separately per theme (`MaguilanoteSettings.colors.light` / `.dark`, both `ThemeColors` — new in `src/types.ts`, defaulted from `DEFAULT_THEME_COLORS`); switching the Theme dropdown swaps which theme's color set is shown/edited, defaulting to the same hex values for both themes except "Default" card background (white on light, dark on dark, as before). Card color presets now resolve through `--mgn-card-color-<key>` CSS variables (previously fixed hex in `CARD_COLORS`), kept in sync with the active theme by `BoardView.applyAppearance()`.
  - Fixed the Draw/Sketch pen-color picker, which had been reusing `CARD_COLORS` (now theme-variable-based) as its swatch list — pen ink is drawn once and persisted as plain hex in the board file, and must not silently repaint if the user later changes their card-color theme settings. It now uses its own fixed `DRAW_SWATCHES` hex list, unaffected by theme customization.
  - Fixed the per-color arrowhead `<marker>` fill, which set the CSS-variable-based color via the SVG `fill` *attribute* (unreliable for `var()`) instead of inline `style.fill` (reliable) — arrows in a non-"Default" card color could fail to pick up the right tip color.
  - Fixed the To-do card's checkbox and "+ add item" placeholder always looking dark regardless of the board theme: the checkbox was an unstyled native `<input type="checkbox">` (themed by Obsidian's real app theme, not our board theme) and `.mgn-todo-add`'s placeholder text had no explicit color rule (falling back to Obsidian's default placeholder color). The checkbox is now a custom `currentColor`-based circle (filled when checked) and both `.mgn-todo-add`/`.mgn-todo-title` placeholders now match the existing note-placeholder look (`color: inherit; opacity: 0.55; font-style: italic`), so both follow the card's own color like the rest of its text.
  - Files: `src/board-view.ts`, `src/types.ts`, `src/main.ts`, `src/settings-ui.ts`, `styles.css`.

## 2026-07-09 (follow-up)

- Follow-up fixes to the Settings panel added earlier the same day, still v0.4.0 (no separate bump):

  - The Settings panel's **Board** section (snap to grid, grid size, default note width, templates folder) now exists and is identical to, and stays in sync with, Obsidian's native Settings → Maguilanote tab — both now render from one shared `renderSettingsUI()` (new `src/settings-ui.ts`) instead of two independent copies that had drifted apart.

  - Card colors: merged the "White" and "Dark" presets into a single **Default** preset that resolves via CSS variables (`--mgn-card-default-bg/fg`) to white in the light theme and dark in the dark theme, instead of being two separate fixed-color entries. Boards saved with the old `"white"`/`"dark"` keys keep rendering correctly — `colorOf()` aliases both legacy keys to `"default"`.

## 2026-07-09

- Added an in-canvas **Settings** panel (v0.4.0, MINOR — new feature). A gear icon next to the breadcrumb trail (top-left) opens `SettingsModal` (`src/modals.ts`), with two sections: **Customization** (text-size scale, font, dark/light theme) and **Shortcuts** (every rebindable keyboard shortcut, click-to-record, reset-to-default, plus a read-only mouse/gesture reference table). Files: `src/types.ts`, `src/main.ts`, `src/modals.ts`, `src/board-view.ts`, `styles.css`, `package.json`, `manifest.json`, `docs/*`.

## 2026-07-08

- Fixed two line-selection bugs from the multi-select rework below. (1) Clicking a line to select it never called `e.preventDefault()` on the pointerdown (every other interactive branch — card move, connector, resize, endpoint handle — already did). Without it, the browser's default mousedown action blurred `viewportEl` right after our handler ran, so a subsequent Delete keypress never reached the view's keydown listener — the line looked selected (blue highlight) but Delete silently did nothing. (2) Rubber-band selection required **both** endpoints of a line to fall fully inside the drag rectangle, unlike cards (which select on any overlap) — in practice a marquee almost never fully enclosed a line, so drag-select felt like it didn't work for lines at all. Replaced with a proper segment/rectangle intersection test (Liang-Barsky clip), so a line is selected as soon as any part of it touches the band, matching card behavior. Files: `src/board-view.ts`.
- Line tool follow-ups: Free/Smart routing modes, curve handle, multi-select, rubber-band and Delete-key support (PATCH-ish but bundled as part of the still-unreleased v0.3.0 Line feature above — no separate version bump). Toolbar tooltips for drag-only tools dropped the "— drag onto the board" suffix (redundant now that clicking shows the same hint). New lines and card-to-card arrows default to `arrow: true` (previously the standalone Line tool defaulted the arrowhead off). Edge selection changed from a single `BoardView.selectedEdge: string | null` to `selectedEdges: Set<string>`, so rubber-band selection and Delete now work on one or more lines at once. Board file format: new optional `Edge.mode` (`"free" | "smart"`, undefined = `"smart"` for backward compatibility; new lines are created with `"free"`) and `Edge.bend` (`{x,y}` world point) — a Free-mode line renders as a straight segment clipped to each end's boundary, optionally bowed through a draggable midpoint handle into a quadratic curve; Smart mode is the original auto-routed bezier. Right-click menu on a line gained a "Switch to Smart routing" / "Switch to Free line" toggle. Files: `src/types.ts`, `src/render.ts`, `src/board-view.ts`, `src/modals.ts`, `styles.css`, `docs/*`.
- Reorganized the main toolbar and added a standalone **Line** tool (v0.3.0, MINOR — new feature). The toolbar now has three groups separated by dividers: **draggables** (Note, To-do, Line, Column, Board, Swatch, Sketch, Comment), **flexible tools** (Image, Vault file, Link — all drag-only — and Draw, click-only), and **actions** (Undo, Redo, Shortcuts). Draggable tools are now **drag-only**: a plain click no longer creates anything — the button shakes and shows a hint tooltip prompting a drag. Their single-key shortcuts (`N`/`T`/`C`/`L`/`B`/`S`/`M`) were removed; only `D` (Draw) and `/` (Shortcuts) remain. The old two-click **connect mode** button (`spline`, `A`) was removed — the Line tool replaces it: drag Line onto the board to drop a free segment, then drag either endpoint handle onto a card to anchor it (drop on empty canvas to leave it free). The card-hover blue dot still draws an arrow, and now dropping it on empty canvas leaves a free endpoint too. Board file format: `Edge.from`/`Edge.to` are now optional and a new optional `Edge.fromPt`/`Edge.toPt` (`{x,y}` world point) stores a free endpoint — backward-compatible (existing edges keep both anchors). Files: `src/types.ts`, `src/render.ts`, `src/board-view.ts`, `src/modals.ts`, `styles.css`, `package.json`, `manifest.json`, `versions.json`, `docs/*`.
- Added **Draw** and **Sketch** freehand drawing (v0.2.0, MINOR — new feature). Draw (toolbar, `D`) enters a board-wide draw mode: the main toolbar dims and a reusable **contextual toolbar** (pen / select / eraser | color / stroke size | undo / redo | discard / save) appears beside it, over a dimmed scrim that makes board nodes inert. Strokes support pen pressure (`PointerEvent.pressure`). On save, nearby strokes are clustered into `drawing` board items (gap threshold `DRAW_GROUP_DISTANCE`, tunable in `src/types.ts`); double-clicking a drawing re-opens it for editing. Sketch (toolbar) creates a `sketch` card whose double-click opens a popup canvas (like the vault-file preview) to draw inside; the card shows a scaled preview. Board file format: two new `Item.type`s (`drawing`, `sketch`) and an optional `Stroke[]` `strokes` field (points in item-local coords) — backward-compatible. New dependency `perfect-freehand@1.2.3` (stroke outlining, bundled). Files: `src/types.ts`, `src/draw.ts` (new), `src/board-view.ts`, `src/render.ts`, `styles.css`, `package.json`, `manifest.json`, `versions.json`, `docs/*`.

- Added a versioning policy: SemVer 2.0.0, where only shipping-plugin changes (`src/**`, `styles.css`, user-facing `manifest.json` fields) bump the version and docs/tooling/dev-dependency changes do not. Documents MAJOR/MINOR/PATCH rules, the pre-1.0.0 exception, the Obsidian strict-`x.y.z` constraint on `manifest.json`/`versions.json`, and the three version files to keep in sync. Files: `.claude/instructions/versioning.instructions.md` (new), `.claude/CLAUDE.md`.
- Added optional auto-copy step to `npm run build`: after a successful build, `scripts/copy-to-vault.mjs` copies `main.js`, `manifest.json`, `styles.css` into a vault plugin folder read from a gitignored `.env.local` (`OBSIDIAN_PLUGIN_DIR=...`). No-op if `.env.local` is absent. Files: `scripts/copy-to-vault.mjs`, `package.json`, `.env.local` (untracked), `docs/DEVELOPMENT.md`.
- Fixed `docs/DEPENDENCIES.md`: pinned-version table and esbuild CVE note were stale (`package.json` had already moved to `esbuild@0.28.1`/`typescript@5.4.2` but the doc still said `0.20.0`/`5.4.0`). Updated table and dropped the now-inapplicable CVE mitigation reasoning since `0.28.1` is already >=0.25.0. Files: `docs/DEPENDENCIES.md`.
- Refactored `styles.css` to centralize repeated colors as CSS custom properties (`--mgn-*`) in a `:root` block, instead of the same hex/rgba values being copy-pasted across the toolbar, zoombar, search bar, crumbs, edges, and card states. Also fixed a visual inconsistency where the "Dark" card color preset looked different between note/comment cards and to-do cards: note placeholders and code/blockquote styling used hardcoded black-based `rgba()` values (readable only on light cards), while to-do inputs already used `color: inherit` + `opacity`. Switched the note-side rules to the same `inherit`/`color-mix(currentColor, ...)` approach so all card types stay legible across every color preset. No behavior change beyond the corrected contrast. Files: `styles.css`.
- Added line (arrow/connector) color. Right-click a line -> "Line color" submenu, reuses the same 10-color palette as cards. Stored as `Edge.color` (board file format addition, backward-compatible since it's optional). Files: `src/types.ts`, `src/render.ts`, `src/board-view.ts`.
